# MECE 优化示例

## direct-result 样例

输入特点：
- 用户问题完整
- 明确要求直接给结果

理想输出特征：
- 直接给 `推荐维度`
- 只展示被选中的拆法，不罗列方法菜单
- `MECE 检验` 和 `优先级建议` 要保留，但保持简洁自然

## missing-info 样例

输入特点：
- 用户只说“帮我分析一下这个项目”

理想输出特征：
- 只补 1-2 个关键缺口
- 问法带轻量脚手架，例如“如果你最卡的是获客、交付或复购，直接告诉我是哪一段”
- 不出现方法菜单或大表格表演

## staged 样例

输入特点：
- 用户说“帮我一步步拆”

理想输出特征：
- Step 1 先给 `推荐维度 + 初步类别 + 边界提醒`
- 暂停语自然
- 不提前给完整分解表，也不罗列备选方法菜单
