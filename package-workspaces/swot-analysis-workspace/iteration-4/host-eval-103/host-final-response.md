**SWOT 分析**

分析对象：面向大学生的面试准备小项目（早期验证阶段）

**四象限要素识别**

| 优势（S） | 劣势（W） |
|---|---|
| 已有 `3` 个愿意试用的真实种子用户，说明不是纯空想 | 预算少，获客、内容制作、技术投入都受限 |
| 离用户近，容易快速收集反馈并迭代 | 品牌弱，用户初期不容易信任“新项目” |
| 聚焦“大学生面试”这个具体场景，比泛求职产品更容易切入 | 很难一开始就覆盖简历、笔试、群面、单面、行业题库等全链路 |
| 小项目决策快，试错成本低 | 若依赖人工辅导，交付效率和利润率都偏低 |

| 机会（O） | 威胁（T） |
|---|---|
| 大学生求职焦虑强，愿意为“更有把握的面试结果”尝试工具 | 竞争强，大平台、AI 面试工具、免费内容都很多 |
| 用户需求并不只是“知识”，而是“陪练、反馈、个性化改进” | 免费替代品多，用户付费意愿容易被压低 |
| 可从垂直切口进入，如“互联网实习”“银行秋招”“英文面试”等 | 大厂或成熟平台可以更快复制功能 |
| 校园传播链条短，若体验好，熟人推荐效率高 | 面试效果难量化，口碑一旦不好会影响早期扩散 |

**要素优先级**

最关键优势：你已经有可触达的种子用户和真实反馈入口。  
最致命劣势：预算少，导致你不能靠投放解决增长，也不能一开始做大而全。  
最宝贵机会：大学生需要的不是信息堆砌，而是针对性练习和反馈，这里仍有产品空间。  
最严重威胁：竞争者很多，且免费内容和通用 AI 会迅速稀释你的差异化。

**交叉战略分析**

**SO 战略（用优势抓机会）**
- 先围绕这 `3` 个试用者做深，不做广，找到一个高频痛点，如“不会回答八股/不会自我介绍/不会结构化表达”。
- 利用离用户近的优势，把产品做成“练一次就能看到改进建议”的强反馈体验，而不是资料库。
- 从一个细分人群切入，例如“找实习的大学生”或“互联网岗求职者”，先建立局部口碑。

**WO 战略（补短板抓机会）**
- 因为预算少，不要做全流程平台，先做一个最小可用模块：如“模拟面试 + 回答点评”。
- 因为品牌弱，优先积累可展示的用户结果，如前后回答对比、录音点评样例、试用反馈。
- 因为资源有限，优先选择低成本获客方式：同学转介绍、校园社群、求职交流群，而不是广告投放。

**ST 战略（用优势对抗威胁）**
- 竞争强时，不和大平台拼“功能多”，而是拼“更懂某类学生的具体场景”。
- 免费替代品多时，不卖信息，卖“个性化反馈”和“真实演练感”。
- 大平台可复制功能时，你要用更快的迭代速度和更近的用户关系，持续优化细节体验。

**WT 战略（降低短板与风险叠加）**
- 不要一开始做 App、课程体系、题库社区三合一，这会同时放大预算和竞争压力。
- 不要过早定低价年费套餐，先验证用户是否愿意为单次服务或轻量订阅付费。
- 不要依赖“大家都需要面试准备”这种大叙事，必须收窄人群和场景，否则会被泛产品淹没。

**战略优先级建议**

优先执行：`SO + WO` 组合战略，也就是“先聚焦细分痛点做出强体验，再用低成本方式验证付费和传播”。

建议你接下来 2-4 周只做三件事：
- 做一个单点 MVP：只解决一个问题，比如“自我介绍/常见面试题回答优化”。
- 用这 `3` 个试用者跑完整闭环：试用前问题、使用过程、使用后是否觉得更有信心。
- 验证最小付费信号：哪怕只是 `9.9` 或 `19.9` 的轻付费，也比“口头说愿意用”更有价值。

**关键提醒**
- 你现在最大的风险不是“竞争强”，而是“产品边界不清”。很多小项目不是输给竞品，是输给一开始做太散。
- 早期阶段最重要的不是用户数，而是能不能证明一个细分场景里，用户用了之后明显变好。
- 如果 3 个试用者里有 2 个愿意继续用、愿意推荐、甚至愿意小额付费，这个项目就值得继续。
- 如果试用反馈分散、需求很杂，说明你还没有找到真正的切口，应继续缩窄，而不是扩功能。
